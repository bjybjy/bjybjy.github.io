维度的降低

1.PCA的理解

提起维度的降低，应用最多的便是PCA。对PCA的解释有两种：1种是最大化投影后数据的方差，因为方差最大的时候，数据的信息丢失的最少。第2种是最小化投影误差。2种方法最终都归结到求解到协方差矩阵的特征向量。

PCA的一般处理流程：

  • 1.将A的每一列按均值做归一化，使归一化后的A的每一列均值为0；
  • 2.求出A的协方差矩阵的特征值以及特征矩阵V。
  • 3.B = AV

PCA的一个优势是可以对变换后的样本数据进行重构只要保留特征矩阵就可以了。

2. K值的选择

一种确定K值的方法是K个特征值与原来D个特征值之和>=0.99或者0.95

3.对PCA的评价

  • 虽然能够降低维度，但是PCA不适合用来阻止过拟合，因为PCA没有考虑标签数据，只是考虑了样本数据的。
  • 一般求解方法使用SVD，速度更快。而且SVD没有计算协方差矩阵，对于某些矩阵，求协方差时很可能会丢失一些精度。
  • 一般情况下复杂度为O(D^3),但有一些学者的方法能达到O(MD^2)

    4.Kernel PCA

    传统的PCA只是利用线性的特征组合，如果想找到非线性的特征组合的话就要考虑Kenel PCA，它可以利用一些非线性核的性质。关于kernel PCA可参看kernel PCA。

    5.对kernel PCA的评价

  • Kernel PCA需要计算的矩阵的特征是NxN阶的，而线性PCA为DxD阶，对于高维数据，kernel trick依然可以使维度降至N
  • Kernel PCA能够提取出非线性的特征，具有更大的灵活性

推荐系统：Introduction

1、CF算法的优缺点：

  • 不用处理物品的特性信息（以书为例，比如说书的种类、作者等）、信息不用进入推荐系统进而对其进行维护

2、Content based：

  • 基本思路：抽取物品信息特征和用户行为模式和反馈来构建特征
  • 不需要大量的用户历史记录
  • 没有冷启动问题
  • 对物品的一些主观性的评定（好用，设计得很好等）很难自动获取。因此，将这些评定放进系统是一个很费时而且容易出错的过程

3、基于知识的推荐：

  • 对于有很多 one-time buyer的商品领域（比如电子产品），CF和Content based方法不适用，对于CF来说，用户相似性矩阵的构建是一个问题，而对于Content-based来说，用户的行为模式和反馈几乎没有
  • 可以加入认为构建专业的特征，比如相机的分辨率、价格
  • Constraint-based recommenders是基于知识推荐里常见的例子，以相机为例，其基本思路是提供分辨率、重量，价格等过滤信息，列出符合条件的物品。但这种方法有一个缺点是：用户的个性化丢失了，因为对于不同的用户，推荐列表可能完全一样。一个解决办法是可以询问用户分辨率和重量你更看中哪一个？
  • 基于知识推荐的另一个思路是可以考虑和用户进行互动。以CF为例，和用户的交互可能只是让用户对某个物品评分，或者对摸个物品做出喜欢或者不喜欢。对于基于知识的推荐可以考虑询问用户所要商品的最大价格、重量等，但这种方法有一个缺点是对询问特征的选择有要求，需要有相关人员的理解。同时，对于物品的特征维度很大的情况，这种询问的过程可能会很繁琐。一种改善的方法是设计对话系统，从用户的回应中来推断用户的需求和喜好。

4、混合推荐方法

  • 融合CF方法和基于内容推荐的方法解决冷启动问题

最小生成树MST

问题的定义是给定一幅无向权重图，寻找出其生成树中权重最小的那一棵

原始做法

根据切分定理，在每一次切分中切分边的权重最小的那条边一定位于最小生成树中，只要一直切分找到V-1条有效的切分边即可

Kruskal算法

Kruskal是一种贪心算法，一种思路是每次都查找最小权重的那条边，但是有一个条件，新加入的边不能与已经查找到生成树中的边构成环（即需要满足生成树的要求）

Prim算法

Prim算法亦是一种贪心算法，他是要求新加入树种的节点到生成树的距离最小，一般的思路是维持一个优先队列，每次去visit一个节点，如果节点没有访问过，同时如果原先到生成树的距离大于该点的权重，那么就插入优先队列

SP(最短路径）

单源最短路径问题：给定一个有向图，求从一个起点到终点的最短路径
如果不考虑权重，最短路径的问题可以用BFS解决。但如果包含权重分为3种情况：（1）非负权重的Djiksla算法其复杂度为O（ElogV）（2）DAG（有向无环图）topilogical sort复杂度O（V+E）（3）除了含有负权重的环，BellMan-Ford含有非负权重的环

Dijkstra SP

与Prim算法类似，其区别是Prim算法寻找的是离树最近的顶点，而Dijkstra算法寻找的是离起点最近的点，除此之外两者几乎一直

Topological SP

其基本思路是利用Topoloical排序的顺序去relax节点

Bellman-Ford SP

Naiive method：relax 每一条边，重复V轮EV
If distTo[v] does not change during pass i, no need to relax any edge pointing from v in pass i+1.
因此可以维护一个队列，里边放进去在该轮中，distTo的值改变的那个顶点

特征的选择

根据特征选择的形式又可以将特征选择方法分为3种：

  • Filter：过滤法，按照发散性或者相关性对各个特征进行评分，设定阈值或者待选择阈值的个数，选择特征。
  • Wrapper：包装法，根据目标函数（通常是预测效果评分），每次选择若干特征，或者排除若干特征。
  • Embedded：嵌入法，先使用某些机器学习的算法和模型进行训练，得到各个特征的权值系数，根据系数从大到小选择特征。类似于Filter方法，但是是通过训练来确定特征的优劣。摘自知乎

1.Filter(过滤法）

  • 方差选择法：计算一列特征的方差，当方差大于阈值时，选用该特征。
  • 相关系数法：计算每一个特征与标签数据的相关系数选择K大的
  • 互信息法：通过信息学的Mutul-Info来选择特征与标签相关性最大的

2.Wapper(封装法）

比较长用的是Recursive Feature elimination

  • 通过递归消除特征法使用一个基模型来进行多轮训练，每轮训练后，消除若干权值系数的特征，再基于新的特征集进行下一轮训练。
  • 同样可以反过来进行操作，刚开始特征集合为空，每一轮训练增加一个特征。

3.Embedded（嵌入法）

  • Lasso：lasso能够产生稀疏解，因此对特征选择具有重要意义。但对于两个近似共线的特征lasso往往任意选择一个，但近似共线的特征重要性可能不同。因此，lasso算法并不如ridge方法稳定。相应的解决办法是可以考虑elastic net，即把lasso与ridge结合。
  • Random Forest：在随机森林中某个特征X的重要性的计算方法如下：
    （1）对于随机森林中的每一颗决策树,使用相应的OOB(袋外数据)数据来计算它的袋外数据误差,记为errOOB1.
    （2）随机地对袋外数据OOB所有样本的特征X加入噪声干扰(就可以随机的改变样本在特征X处的值),再次计算它的袋外数据误差,记为errOOB2.
    （3）假设随机森林中有Ntree棵树,那么对于特征X的重要性=∑(errOOB2-errOOB1)/Ntree,之所以可以用这个表达式来作为相应特征的重要性的度量值是因为：若给某个特征随机加入噪声之后,袋外的准确率大幅度降低,则说明这个特征对于样本的分类结果影响很大,也就是说它的重要程度比较高。

知道特征的重要性之后，就可以用来做特征选择随机森林OOB特征选择：
(1) 首先建立m棵决策树，然后分别计算每棵树的OOB袋外误差errOOBj。
(2) 计算特征xi的重要性。随机的修改OOB中的每个特征xi的值，再次计算它的袋外误差errOOBi；xi的重要性=(∑errOOBi−errOOBj)/Ntree
(3) 按照特征的重要性排序，然后剔除后面不重要的特征；
(4) 然后重复以上步骤，直到选出m个特征。

数据缺失问题的处理

数据缺失问题是机器学习中经常碰到的问题，主要的类别有：

  • 完全随机缺失（missing completely at random,MCAR）：指的是数据的缺失是完全随机的，不依赖于任何不完全变量或完全变量，不影响样本的无偏性。如家庭地址缺失。
  • 随机缺失(missing at random,MAR)：指的是数据的缺失不是完全随机的，即该类数据的缺失依赖于其他完全变量。例如财务数据缺失情况与企业的大小有关。
  • 非随机缺失(missing not at random,MNAR)：指的是数据的缺失与不完全变量自身的取值有关。如高收入人群的不原意提供家庭收入。

对于随机缺失和非随机缺失,删除记录是不合适的,随机缺失可以通过已知变量对缺失值进行估计；而非随机缺失还没有很好的解决办法。
解决上述问题的方法主要有三大类：删除元祖、数据补齐、不处理。

1.删除元祖

简单来说，也就是将存在遗漏信息属性值的对象（元组，记录）删除，从而得到一个完备的信息表。这种方法简单易行，在对象有多个属性缺失值、被删除的含缺失值的对象与初始数据集的数据量相比非常小的情况下非常有效，类标号缺失时通常使用该方法。
然而，这种方法却有很大的局限性。它以减少历史数据来换取信息的完备，会丢弃大量隐藏在这些对象中的信息。在初始数据集包含的对象很少的情况下，删除少量对象足以严重影响信息的客观性和结果的正确性；因此，当缺失数据所占比例较大，特别当遗漏数据非随机分布时，这种方法可能导致数据发生偏离，从而引出错误的结论。

2. 数据补齐

  • 可以使用一些统计量，比如数据的平均值，中位数等
  • 可以根据个人对业务的理解人为的去填充，这种方法造成的偏差是最小的。但不适用于数据规模比较大的情况。
  • 新增一列特征，是否为缺失值，该种方法能够完整的保证数据的完整性，但同时会增加样本数据的维度。而且对于数据量不大的情况下，效果往往不好。
  • 就近补齐，考虑采用就近的数据进行补齐。这个方法的缺点在于难以定义相似标准，主观因素较多。
  • K最近距离邻法（K-means clustering）先根据欧式距离或相关分析来确定距离具有缺失数据样本最近的K个样本，将这K个值加权平均来估计该样本的缺失数据。
  • 线性插值回归（Regression）基于完整的数据集，建立回归方程。对于包含空值的对象，将已知属性值代入方程来估计未知属性值，以此估计值来进行填充。当变量不是线性相关时会导致有偏差的估计。
  • 期望值最大化方法（Expectation maximization，EM）EM算法是一种在不完全数据情况下计算极大似然估计或者后验分布的迭代算法。在每一迭代循环过程中交替执行两个步骤：E步（Excepctaion step,期望步），在给定完全数据和前一次迭代所得到的参数估计的情况下计算完全数据对应的对数似然函数的条件期望；M步（Maximzation step，极大化步），用极大化对数似然函数以确定参数的值，并用于下步的迭代。算法在E步和M步之间不断迭代直至收敛，即两次迭代之间的参数变化小于一个预先给定的阈值时结束。该方法可能会陷入局部极值，收敛速度也不是很快，并且计算很复杂。
  • 通过寻找属性间的关系来对遗失值填充。它寻找之间具有最大相关性的两个属性，其中没有遗失值的一个称为代理属性，另一个称为原始属性，用代理属性决定原始属性中的遗失值。这种基于规则归纳的方法只能处理基数较小的名词型属性。

在对不完备信息进行补齐处理的同时，我们或多或少地改变了原始的信息系统。而且，对空值不正确的填充往往将新的噪声引入数据中，使挖掘任务产生错误的结果。

高斯混合模型（Gaussian Mixture Model）

单高斯模型具有一定的局限性，混合高斯模型更高能去拟合样本的分布，混合高斯模型的定义为，对于一个d维的样本，我们假设它是由K个高斯混合而成，每个高斯都有其权重，考虑到概率的因素，这些权重之和为1.

1.基本模型

根据最大似然的原理只需最大化似然函数，但是这个似然函数优化起来有些麻烦因为log函数出现了加法，这样对于求解造成了麻烦，一般情况下可以采用EM算法去求解。

2.EM算法

将问题转化为最大化lower bound, 一般是将问题转化为Lowerbound L(q, theta)和KL距离KL(q||p).我们知道KL距离是恒大于0的(根据jesen不等式）那么在E步，由于固定了参数theta，那么由于目标函数不含theta，故保持不变，要使下界最大，只需KL最小即可。当去q（z）=p（z|x，theta）使，KL最小为0；在M step， 保持q（z）不变，优化参数theta，得到新的theta，theta的改变使得KL部分恒大于0，对于Lower bound也肯定增大，因为我们就是根据lower bound来求解theta的

3.EM算法的评价

  • EM算法速度很慢，迭代次数多，一般的做法是利用KMeans确定好初值，再进行训练。
  • EM算法是非凸优化，没有全局最优解

    4.LDA（理解的一般）

  • Bayesian观点认为参数不是一个固定值，而使服从某种分布，样本是固定的，我们需要做的就是推断出参数的分布，而频率观点是认为样本是随机的，而参数是固定的，但是未知的，我们要做的是求出这个值
  • LDA基本思想是对于一篇文章被选中的概率P(di)，P(tk|di)即在文档di的条件下tk的概率，P(wl|tk）即在主题tk的条件下词语wl出现的概率。而我们要做的就是最大化分布P（W|D）。在这个当中我们又加了2个超参数，用来估计两个分布

Linux常用命令

主要命令如下:

  • ls/ls -r:list directory content列出当前所有文件和文件夹的名称，加上-r之后是文件和文件夹的详细信息
  • dir [文件夹名称]:是dos下的命令，与ls命令类似
  • chmod：用于改变指定文件的权限命令
    Linux文件的三种身份和四种权限，三种身份分别为：（1）u：文件的拥有者（2）g：文件所属的群组（3）o：其他用户
    对于每个身份，又有四种权限，分别为：
    (1)r：读取文件的权限（read） 常用数字4表示
    (2)w：写入文件的权限（write） 常用数字2表示
    (3)x：执行的权限（execute） 常用数字1表示
    (4)s：特殊权限
    举例：
    chmod 751 test.txt 对文件的拥有者执行4+2+1的权限，对文件所属的群组执行4+1的权限，对其他用户执行1权限
    chmod u=rwx,g+rx,o-x test.sh +和-相当于增加和删除一些权限
    chmod a+x test.sh a相当于所有的3种用户u、g、o，即对所有的三种用户赋予执行的权限
    drwxr-xr-x 一共有10个字符，第一个字符指是不是普通文件，第2~4代表文件拥有者，第5~7代表文件所属的群组，第8~10代表其他用户
    • Chown：用来更改某个目录或文件的用户名和用户组
      chown HEHE hh.c ：修改文件hh.c为HEHE用户所有
      chown -R HEHE:net /tmp/sco：将目录 /tmp/sco 这个目录的所有者和组分别改为HEHE和组net
    • mkdir：-m 用来设置权限模式，默认是0777，可用chmod更改；-p 用来创建多级目录；-v 显示创建新目录时的信息
    • Rmdir：用于删除空的目录 -v选项：提示；-p选项：级联删除
      rmdir dir1 dir2:删除dir1和dir2两个空目录
      rmdir -p -v 1122/1:假设1122/1和1122/2两个目录，先删除1122/1,再删除1122出错，因为1122不为空
    • rm：删除文件或者目录 -f选项：不管目录下的文件存在不存在，都不给予提示；-r：删除此目录下的所有文件，但是此目录不删除；-rf组合删除:删除该目录以及该目录下的所有文件； -i：删除文件或文件夹前须要确认
    • touch：创建一个新的文件 -a：改变访问时间；-m：改变改动时间；-t timestamp：改变访问时间和改动时间为timestamp
    • Vi:创建一个新的文件，进入vi后，点击字母i进行编辑模式，若输入错误必须先按Esc退出该模式，进入命令行模式，按字符x进行删除。保存时需要在命令行模式下进行：（：w filename退出以filename名称保存；：wq保存修改并退出vi； ：q! 不保存强制退出）
    • mv：移动文件到新的位置 -b ：若需覆盖文件，则覆盖前先行备份；-f ：force 强制的意思，如果目标文件已经存在，不会询问而直接覆盖；-i ：若目标文件 (destination) 已经存在时，就会询问是否覆盖；-u ：若目标文件已经存在，且 source 比较新，才会更新(update)；-t：即指定mv的目标目录，该选项适用于移动多个源文件到一个目录的情况，此时目标目录在前，源文件在后。
    • cp：复制文件/文件夹到新的位置 （默认情况下，cp命令不能复制目录，如果要复制目录，则必须使用-R选项；当“源文件”为多个文件时，要求“目标文件”为指定的目录。
      cp file /usr/men/tmp/file1：复制文件file到file1
      cp -r /usr/men /usr/zh:将目录/usr/men下的所有文件及其子目录复制到目录/usr/zh中
      cp -i /usr/men m*.c /usr/zh:交互式地将目录/usr/men中的以m打头的所有.c文件复制到目录/usr/zh中
    • cat、more、less的区别：cat显示全部的文件内容。more和less都可以显示一部分内容进行滚动浏览，但区别是less可以利用pageup和pagedown键进行前后浏览，而more只能通过空格键往后浏览。后两者按q键可以退出
      cat -n: 显示行号 cat -b: 显示行号且忽略空行
    • head/tail:可以显示文件前n行和后n行的数据
      例：head -n filename：显示前n行的文件内容
    • find 指定的文件：查找相关文件
    • Whereis：查找指定的文件源和二进制文件和手册等（例whereis java:查找java的安装地址)
    • which:用于查询命令或别名的位置。
    • Grep：查找文件里符合条件的字符串。Grep “sj” filename在文件filename中查找是否有字符“sj”
    • Tar：用于多个文件或目录进行打包，但不压缩，同时也用命令进行解包
      tar -c 创建新文档 -x 解包 -v 显示正在处理的文件名 -f 取代默认的文件名
      tar -cvf project.tar project/* 把project目录下所有文件打包成文件project.jar
    • Su：切换用户命令
    • Sudo：一系统管理员的身份执行命令
    • Passwd：用于修改用户的密码
    • pwd：显示当前用户所在工作目录位置
    • Who：显示系统中有那些用户在使用。
    • man：帮助命令，man ls
    • Free：查看当前系统的内存使用情况
    • Uptime：显示系统运行了多长时间
    • Ps：显示瞬间进程的动态
    • Top: 动态地显示进程
    • Date：显示或设定系统的日期与时间。
    • Last：显示每月登陆系统的用户信息
    • Kill： 杀死一些特定的进程
    • Logout：退出系统
    • Useradd/userdel:添加用户/删除用户
    • Clear：清屏
    • Passwd：设置用户密码
    • Arp：网络地址显示及控制
    • Netstat：显示网络连接.路由表和网络接口信息
    • wc: 计数 -l: 行数；-w: 字数；-c: 字符数；
      wc -l file1 file2 ……可以统计多个文件
    • file 察看文件类型
    • fg：把后台进程放到前台。fg %1 把后台第一个作业放到前台
    • bg：把前台进程放到后台
    • jobs：显示后台或挂起的进程
    • ps：显示所有进程 ps -f 显示完全信息，包括占用cpu时间，开始时间。
    • kill -9 324 强制结束进程标识符为324的进程
    • sort 排序 -r 逆序；-d 字典顺序
      ls | sort -r：逆序列出文件名
    • tr 字符替换 -d 删除指定字符；ls | tr -d ‘log’；ls | tr ‘d’ ‘g’把d变成g
    • crontab命令的功能是在一定的时间间隔调度一些命令的执行。任务调度的crond常驻命令crond 是linux用来定期执行程序的命令。当安装完成操作系统之后，默认便会启动此任务调度命令。crond命令每分锺会定期检查是否有要执行的工作，如果有要执行的工作便会自动执行该工作。
    • crontab命令选项:-u指定一个用户;-l列出某个用户的任务计划;-r删除某个用户的任务;-e编辑某个用户的任务
      分 小时 日 月 星期 命令
      0-59 0-23 1-31 1-12 0-6 command (取值范围,0表示周日一般一行对应一个任务)
      30 6 0 ls 指定每星期日的6:30执行ls命令[注：0表示星期天，1表示星期1
    • 环境变量 gedit /etc/proifle 更新环境变量source /etc/profile
